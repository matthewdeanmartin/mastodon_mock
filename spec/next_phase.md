## 1. Current state (as of v0.1.0)

A stateful Mastodon API mock built on **FastAPI + SQLAlchemy 2.0 (sync) + SQLite**,
driving Mastodon.py's read/write surface. Writes persist and are reflected in
subsequent reads.

### What's implemented and verified

- **Full app factory** (`mastodon_mock/app.py`): config load ŌåÆ engine ŌåÆ `create_all`
  ŌåÆ seed ŌåÆ routers ŌåÆ `/media` static mount.
- **15 routers** under `mastodon_mock/routers/`: `oauth`, `instance`, `accounts`,
  `statuses`, `timelines`, `relationships`, `notifications`, `media`, `search`,
  `lists`, `favourites_bookmarks`, `filters`, `polls`, `preferences`,

  intentionally unrouted (Mastodon.py gets a 404).
- **Serializers** (`mastodon_mock/serializers/`) producing Mastodon-shaped JSON.
  Note: `AttribAccessDict` fills missing annotated fields with `None`, so serializers

- **Alembic**: `alembic.ini` + `mastodon_mock/alembic/` with an autogenerated
  initial migration matching the models.
- **CI/release workflows** (`.github/workflows/`): `build.yml` (quality gates),
  `create_draft_release.yml`, `release.yml` (bumpŌåÆbuildŌåÆPyPI publish via trusted

### Test harness facts (important for the next bot)

- Contract tests need a **real running server** (Mastodon.py uses `requests`, not
  ASGI transport). `tests/conftest.py` spins up `uvicorn` on an OS-assigned free
  port in a daemon thread. **Do NOT try to use `httpx.ASGITransport` with a sync
  client** ŌĆö it fails (`handle_request` vs `handle_async_request`). This was hit
  during initial development; use the live-server fixture or an async httpx client.
- The vendored `Mastodon.py/` clone is **gitignored** and is NOT the test
  dependency ŌĆö `mastodon.py` is installed from PyPI (dev group). Keep the clone as
  a reference for re-running the endpoint-inventory greps when Mastodon.py updates.

## 2. Deliberate simplifications / things to know before changing code

These are intentional per the spec ŌĆö don't "fix" them without reason:

- **Scope enforcement is OFF by default.** `oauth_tokens.scopes` is stored and
  echoed; checked only when `config.auth.enforce_scopes = true` (P2, now built ŌĆö
  coarse read/write by method, see `mastodon_mock/middleware.py`).
- **Rate limiting is OFF by default.** No `429`/`X-RateLimit-*` unless
  `[tool.mastodon_mock.ratelimit] enabled = true` (P2, now built ŌĆö per-token fixed
  window, see `mastodon_mock/middleware.py`).
- **No real OAuth code flow.** `password` and `authorization_code` grants return
  400. User tokens come from seed config or `/api/v1/_mock/login`.
- **No federation / no media processing / no streaming / no admin / no push.**
- **Reblogs are their own `statuses` row** with `reblog_of_id` set (matches
  Mastodon's nested `Status.reblog`).
- **Status edit history** is a JSON column (`statuses.edit_history`), not a table.
- **Idempotency** is a real table (`idempotency`) keyed `(account_id, key)`; there is
  **no TTL** ŌĆö keys live forever within a DB. Fine for tests; note if it matters.
- **Pagination** for favourites/bookmarks paginates by `Status.id` (not the
  favourite/bookmark row id) for cursor consistency ŌĆö slightly off from real
  Mastodon ordering but stable. See `routers/favourites_bookmarks.py`.

## 3. Known gaps / sharp edges (candidate bugs to validate)

These are **not yet covered by a test** and are the most likely places for a
mock-vs-real divergence. Prioritize writing a contract test for each, then fix if
it diverges:

1. **Scheduled-status publication.** Scheduled statuses are listed but never
   converted to real statuses when `scheduled_at` passes (spec allows leaving this
   as Stub). If a consuming suite asserts publication, implement lazy conversion.
2. **`status_reply` mention threading.** `services.parse_mentions` resolves
   `@username` / `@username@domain` against local rows only. Replies that mention
   remote-looking accounts won't notify unless the row exists. Verify reply chains
   + CW/visibility inheritance against real Mastodon.
3. **`account_statuses(only_media=...)`** filters on `MediaAttachment.status_id IS
   NOT NULL` globally in a subquery ŌĆö re-read the query; confirm it scopes to the
   account's own statuses correctly.
4. **Poll expiry & `poll` notifications.** `polls.expired` is never set (no time
   driver); the `poll` notification type is unimplemented (spec marks it low
   priority). 
5. **Domain blocks vs `Relationship.domain_blocking`.** Domain blocks live in their
   own table and are surfaced in the relationship serializer by a per-call lookup ŌĆö
   verify `account_relationships` reflects a domain block correctly.
6. **`Link` header pagination round-trips.** Unit-tested shape exists, but no test
   drives Mastodon.py's `.fetch_next()` / `paginate=` across a real multi-page
   result. Add one (post 25 statuses, page through home timeline).
7. **`update_credentials` field/avatar edge cases.** `fields_attributes[n][...]`
   parsing and avatar/header multipart upload are implemented but only lightly
   tested.

### Bug-hunt round (array `[]` query params)

A coverage round surfaced a class of real mock-vs-real divergences: Mastodon.py
serializes list arguments as `name[]=a&name[]=b` (see `Mastodon.__generate_params`),
but several endpoints bound a plain `name` FastAPI `Query`, so the values were
**silently dropped**. Added `routers/helpers.py::array_query` (reads both `name`
and `name[]`) and fixed:

1. `GET /api/v1/notifications` `types`/`exclude_types` filter (pre-existing).
2. `GET /api/v2/notifications*` `types`/`exclude_types` filter.
3. `GET /api/v1/statuses?id[]=` bulk fetch (returned `[]`).
4. `GET /api/v1/accounts?id[]=` bulk fetch (returned `[]`).
5. `GET /api/v1/accounts/relationships?id[]=` (list form).
6. `GET /api/v1/accounts/familiar_followers` ŌĆö Mastodon.py sends ids in a **JSON
   body** (`use_json=True`); now read from query *or* body.
7. `GET /api/v1/markers?timeline[]=` filter (returned all timelines).

Covered by `tests/test_bughunt_bulk_by_id.py` (6) and validated against the live
server via `tests/integration/test_integration_readonly.py` (bulk statuses).

## 5. How to work in this repo

```bash
# All commands via uv (locked venv). Git Bash is the shell.
uv run pytest -q                      # full suite (in-memory sqlite)
uv run pytest tests/test_contract_core.py -p no:randomly -x   # focused, deterministic
uv run make check-ci                  # CI gate: ruff + black --check + mypy + tests + metadata
uv run ruff check mastodon_mock tests --fix
uv run black mastodon_mock tests
uv run mypy mastodon_mock tests       # strict

uv run mastodon_mock serve --in-memory --port 3000   # manual poking
uv run alembic revision --autogenerate -m "msg"      # after model changes
```

- **Keep `spec/` in sync** with reality. When you move a row StubŌåÆFull in
  `03-api-coverage.md`, update its status note. The coverage table doubles as the
  progress tracker.
- **Every new "Full" endpoint needs a Mastodon.py contract test** (drive through
  `from mastodon import Mastodon`, never raw httpx ŌĆö keeps the "if Mastodon.py
  supports it" contract honest).
- mypy is **strict**; `tests.*` relaxes untyped-def rules. Watch for `Any` leaking
  out of `request.app.state.*` and `form.get(...)` ŌĆö annotate/coerce explicitly.
